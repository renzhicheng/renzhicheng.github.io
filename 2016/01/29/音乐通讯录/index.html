注意点：取值

在TableView中section的使用：
先看NSFetchedResultsController中初始化方法及它的属性sectionNameKeyPath在文档中怎么说的。

-(instancetype)initWithFetchRequest:(NSFetchRequest )fetchRequest managedObjectContext:(NSManagedObjectContext )context sectionNameKeyPath:(NSString )sectionNameKeyPath cacheName:(NSString )name

sectionNameKeyPath

A key path on result objects that returns the section name. Pass nil to indicate that the controller should generate a single section.
The section name is used to pre-compute the section information.
返回的就是section name。

所以在代码中，需要添加key值，singer就是我的key值，CoreData会根据key值对应取出value，非常方便。

属性section中文档有写：The objects in the sections array implement the NSFetchedResultsSectionInfo protocol.执行了NSFetchedResultsSectionInfo协议。

在NSFetchedResultsSectionInfo中可以看到四个属性：numberOfObjects、objects、name、indexTitle。分别是，对象数量、对象、对象名称，索引标题。

注意点：改值

改值，其实思路很简单，将父类的值传给子类，子类原本就有一个判断是否有值，有值的时候修改，没有值得时候新建。这样父类传值后，子类修改在savecontent，在CoreData中可以这里分方便，无需代理，父类reloaddata后，就能直接显示。
